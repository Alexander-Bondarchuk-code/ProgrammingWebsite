Практика: спробуй себе як DevOps Engineer — Рівень 5

Після чотирьох уроків теорії настав час дізнатися, як усе виглядає на практиці. DevOps інженер не просто читає документацію і налаштовує сервери — він автоматизує, контролює, оптимізує. Цей урок — шанс доторкнутися до реальної роботи DevOps через прості, зрозумілі завдання.

Завдання 1: Створи базовий скрипт для автоматизації

Мета: навчитися автоматизувати просту дію.

Приклад: скрипт, який архівує файли певної папки у .zip.

bash Копіювати Редагувати #!/bin/bash zip -r archive.zip /шлях/до/твоєї/папки echo "Архівування завершено" Запусти його через термінал (Linux/Mac) або Git Bash (Windows).

Зрозумій, що таке Bash-скрипт і як він економить час.

Завдання 2: Ознайомся з CI/CD у дії

Мета: побачити, як працює автоматичне тестування та деплой.

Перейди на GitHub → вибери будь-який open source-проєкт.

Перевір, чи в ньому використовується GitHub Actions (у вкладці "Actions").

Подивись:

що саме запускається після кожного оновлення коду (build, test, deploy); як це автоматизовано за допомогою YAML-файлу.

Завдання: коротко опиши, що саме відбувається після коміту в цьому проєкті.

Завдання 3: Створи структуру для простої інфраструктури

Ситуація: уяви, що потрібно запустити невеликий вебсайт на хмарі.

Опиши (словами або у вигляді схеми):

де буде код (наприклад, GitHub),

як він потрапить на сервер (CI/CD),

де буде сервер (наприклад, AWS EC2 або Docker-контейнер),

як користувач відкриватиме сайт (через домен і інтернет).

Це і є архітектурне мислення DevOps інженера.

Завдання 4: Розбери реальний інцидент

Мета: навчитися мислити як DevOps у кризовій ситуації.

Знайди приклад збою великої платформи (наприклад, Google, GitHub, AWS outage). Прочитай короткий опис ситуації: що сталося, чому, як вирішили.

Опиши: як би DevOps міг запобігти цьому в майбутньому (моніторинг, резервне копіювання, масштабування, автоматичне відновлення тощо).

Підсумок

Ці прості вправи — не про глибокі технічні знання. Вони про мислення DevOps: як все пов’язано, що можна автоматизувати, де очікувати проблеми й як її попередити.

DevOps Engineer — це не просто “технар”, а людина-система, яка тримає все під контролем і вчасно реагує. Якщо тобі сподобалось планувати, скриптувати, досліджувати — це може бути твоє. І з цього простого старту можна вийти на одну з найсильніших технічних професій у сучасному ІТ.
